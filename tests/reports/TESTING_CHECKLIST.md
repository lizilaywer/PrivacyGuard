# Word 文档脱敏功能测试清单

## 测试前准备

1. 确保依赖已安装：
   ```bash
   pip install python-docx==1.1.2 mammoth==1.8.0
   ```

2. 准备测试文档：
   - 运行 `verify_word_format.py` 创建测试文档
   - 或使用自己的 Word 文档

---

## 功能测试

### 1. 文件打开测试

- [ ] 打开 `.docx` 文件
  - 检查是否正常显示 HTML 预览
  - 检查信息栏是否显示正确的文档信息（段落数、表格数）
  - 检查 PDF 特有控件是否隐藏（双页模式、黑白遮罩等）
  - 检查替换文本输入框是否显示

- [ ] 尝试打开 `.doc` 文件
  - 检查是否显示"暂未支持"的提示

### 2. 智能扫描测试

- [ ] 点击"智能脱敏"按钮
  - 检查进度条是否正常显示
  - 检查扫描完成后是否显示找到的敏感信息数量

- [ ] 验证识别的敏感信息类型：
  - [ ] 手机号码 (如: 13812345678)
  - [ ] 身份证号 (如: 110101199001011234)
  - [ ] 电子邮箱 (如: test@example.com)
  - [ ] 银行卡号 (如: 6222021234567890123)
  - [ ] 日期时间 (如: 2024年01月01日)
  - [ ] 自定义关键词

### 3. HTML 预览测试

- [ ] 检查 HTML 预览是否正常显示
  - [ ] 文档内容是否完整显示
  - [ ] 敏感信息是否用黄色高亮标记
  - [ ] 表格是否正常显示
  - [ ] 段落格式是否正常

### 4. 自定义替换文本测试

- [ ] 修改替换文本输入框的内容
  - [ ] 输入自定义文本（如 "***"）
  - [ ] 保存文档，检查是否使用了自定义替换文本

### 5. 保存文档测试

- [ ] 保存脱敏后的文档
  - 检查保存对话框是否正常弹出
  - 检查是否成功保存

- [ ] 打开保存后的文档验证
  - [ ] 敏感信息是否被正确替换
  - [ ] 格式是否保持正确

## 已知问题

1. **身份证号正则表达式问题**：
   - 18 位身份证号可能被识别成两个 15 位号码
   - 建议优化正则表达式：`(?<!\d)(\d{6}[12]\d{3}[01]\d[0-3]\d{4}[\dXx])(?!\d)`

2. **跨 run 敏感信息处理**：
   - 当敏感信息跨越多个 run（如部分粗体）时，可能无法完全保持格式
   - 这是 python-docx 的限制

3. **复杂表格格式**：
   - 合并单元格的复杂表格可能需要手动检查
