### 2. 根本原因分析

#### 问题：右键点击删除功能失效

**症状：**
```
✅ 鼠标悬停能变手型（位置检测正确）
❌ 右键点击无响应
❌ contextMenuEvent 未触发
❌ 日志无输出
```

**根本原因：**
```
1. Qt 事件层次结构问题
   └─ QScrollArea 默认拦截某些鼠标事件
   └─ 事件未到达 SinglePageCanvas

2. QLabel 的特殊行为
   └─ 右键菜单处理机制不可控
   └─ 即使设置 PreventContextMenu 也不可靠

3. 事件传播顺序
   └─ 父控件可能优先处理事件
   └─ 子控件无法接收到事件
```

**验证方法：**
```bash
# 分析日志
grep "contextMenuEvent" app_debug.log  # 无输出
grep "右键" app_debug.log               # 无输出
```

**结论：** 事件根本未到达目标控件

### 3. 左键点击重复4次的问题

**原因：事件冒泡**
```
SinglePageCanvas (第一层)
    ↓ 事件未 accept()
QHBoxLayout (第二层)
    ↓ 继续传播
canvas_container (第三层)
    ↓ 继续传播
QScrollArea (第四层)
```

**影响：**
- 性能问题（重复处理）
- 日志混乱（难以调试）

**v19.5 解决方案：**
```python
# 防抖机制
self._event_debounce_ms = 50
if current_time - self._last_event_time < self._event_debounce_ms:
    return False
```

## 🛠️ 解决方案实施

### v19.5 修复方案

#### 方案选择
经过分析，选择了 **事件过滤器（eventFilter）** 方案：

**为什么选择这个方案？**
```
✅ 不改变现有架构
✅ 在最外层捕获事件（不被拦截）
✅ Qt 推荐的事件处理方式
✅ 便于调试和扩展
```

#### 核心实现

**1. 安装事件过滤器**
```python
def __init__(self, ...):
    super().__init__(parent)
    self.installEventFilter(self)  # 监控自己的事件
```

**2. 实现事件过滤器**
```python
def eventFilter(self, obj, event):
    if obj == self:
        if event.type() == event.Type.MouseButtonPress:
            if event.button() == Qt.MouseButton.RightButton:
                self._handle_right_click(event.pos())
                return True  # 阻止事件传播
    return super().eventFilter(obj, event)
```

**3. 统一处理逻辑**
```python
def _handle_right_click(self, click_pos):
    # 删除逻辑（优先手动框，其次 OCR 框）
    # 详细日志输出
    # 发送信号通知主窗口
```

**4. 防抖机制**
```python
self._last_event_time = 0
self._event_debounce_ms = 50

# 防止短时间内重复触发
if current_time - self._last_event_time < self._event_debounce_ms:
    return False
```
