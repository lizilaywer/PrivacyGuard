PrivacyGuard v38 概念稿 B · 保留现有逻辑版

先让人一眼看懂，再让每条真实处理链都保留下来

这版不是为了“看起来高级就把功能砍薄”，而是按当前代码里的真实行为重新排布界面： 单个 PDF 走涂抹工作台，单个 Word 走替换与对比预览，多 Word 直接进入批量文档替换规则模式，多图片继续支持合并成 PDF。 第一眼更简洁，但底层逻辑不混、不藏、不误导。

你这套软件的真实模式，我已经按代码重新对齐 PDF 主要是定位后涂黑/涂白导出。Word 主要是规则替换、智能匹配、手动替换，再导出替换后的文档。两条链分开设计，不再混成一个页面逻辑。
这版重点保留 4 个能力 批量 Word 入口、可隐藏的对比预览、PDF 的黑/白即时切换、多图合并 PDF 与 .doc 转换链路，都会在原型里明确体现。
类比一句话 不是让用户先研究软件，再自己猜该走哪条路；而是像医院前台分诊，文件一进来，软件先把人带到正确窗口。

一上来先“分流”，不先把所有按钮堆满

这一步像医院分诊台，也像打印店前台。用户只要把文件拖进来，系统先判断“这是 PDF 涂抹、Word 替换、批量 Word，还是多图合并”。 这样第一眼看上去会很简洁，但并没有删除功能，只是把复杂度放到正确时机再出现。

为什么这样排，第一眼会更简洁

因为用户第一次并不关心“高级设置里 OCR 框收缩要不要调 3%”，也不关心“PDF 双页按钮在哪”。 用户关心的是：我把文件放进来后，软件带我走哪条路。先分流，后出现相关工具，界面自然就干净了。

和中国办公软件习惯更贴近

中文办公场景更接受“先选文件，再处理，再导出”的顺序，而不是一上来给一排技术控件。 这种做法也更适合政务、律所、法务、行政人员的使用习惯。

通俗解释：为什么这版会更像“现代产品”，但又不偏离你的软件

下面这部分专门用通俗类比讲给小白和产品视角听，不谈术语堆砌，只谈“用户一眼能不能懂、会不会走错、会不会觉得专业顺手”。

通俗类比
按真实逻辑解释
Windows 友好
1. 为什么先做导入分流 像医院前台分诊。病人一进来，不是让他自己研究 CT、抽血、挂哪个科，而是先问“你是什么情况”，然后带去正确窗口。
  • 单个 PDF 去“涂抹窗口”
  • 单个 Word 去“替换窗口”
  • 多个 Word 直接去“批量规则窗口”
  • 多图片去“合并成 PDF 窗口”
2. 为什么 PDF 和 Word 要分成两套工作台 因为它们处理的不是同一类对象。PDF 更像一张已经排好的纸，Word 更像可编辑稿件。
  • PDF 是页面级，重点是遮挡位置和导出不可逆
  • Word 是文本级，重点是替换内容和格式尽量保留
  • 把它们混在一个操作区里，用户会反而更糊涂
3. 为什么对比预览必须保留而且能隐藏 像老师批作业。批改时你想看“原句”和“修改后”的区别；看完了，又希望桌面收干净，只保留主稿。
  • 需要复核时，打开双栏
  • 只想浏览时，收起右栏
  • 这比永远双栏或永远单栏都更灵活
4. 为什么黑 / 白一定要在 PDF 工具栏里 像复印时切黑白模式，不会有人把这个按钮藏进“高级设置”。这是当前任务的即时决定，必须眼前可见。
  • 法律文书公开常用黑色
  • 内部留白版常用白色
  • 用户处理过程中可能反复切换
5. 为什么批量 Word 先规则后执行 像打印店批量套模板。先把规则看清楚，再一键套到整批文件上，出了错还能选择跳过还是停止。
  • 用户看到多个 Word，就知道自己进了批量模式
  • 不会误以为还在处理单个文档
  • 规则模式和执行模式视觉上分明
6. 为什么这版更适合 Windows 用户 因为它不走花哨但不稳的桌面特效路线，而是更强调清晰边界、鼠标命中区、白底高对比、流程直接、标题明确。
  • 更像现代办公工具，而不是网页拼装工具
  • 更适合 125% / 150% 缩放下的阅读和操作
  • 保留系统窗口习惯，对 Win 用户更友好

这一版具体满足了你这次点名的 3 个修改要求

1. 批量 Word 拖入后直接进入文档替换规则模式
2. 对比预览保留，并可隐藏
3. PDF 与 Word 两套逻辑彻底分开

另外我还顺手把当前代码里真实存在的链路也保留进去了：多图片合并 PDF、旧版 .doc 转换、Word 规则导入导出、批量任务停止/跳过、PDF 黑白与单双页可见控制。 也就是说，这版的“简洁”不是减功能，而是减误解、减混乱、减第一次上手的负担。
