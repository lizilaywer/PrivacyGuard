# PrivacyGuard v22.1 修复总结

## 📋 修复历程

### 问题现象
- ❌ 右键点击脱敏框完全无效
- ❌ 控制台没有任何 [DEBUG] 日志输出

### 问题诊断

#### 原始设计 (v22.0)
```python
# 事件过滤器安装在 canvas_container 上
self.canvas_container.installEventFilter(self)

# 检查对象
if obj == self.canvas_container and event.type() == event.Type.MouseButtonPress:
```

**问题**: QScrollArea 的控件层级中，鼠标事件实际到达的是 `viewport()` 而不是 `canvas_container`，因此事件过滤器无法捕获右键点击事件。

### 修复方案 (v22.1)

#### 修复 1: 更改事件过滤器安装位置
```python
# 将事件过滤器安装在 viewport() 上
self.scroll.viewport().installEventFilter(self)
```

#### 修复 2: 更改事件过滤器检查
```python
# 检查对象改为 viewport()
if obj == self.scroll.viewport() and event.type() == event.Type.MouseButtonPress:
```

#### 修复 3: 修复类型不匹配问题
**错误**:
```
TypeError: arguments did not match any overloaded call:
  contains(self, p: QPointF): argument 1 has unexpected type 'QPoint'
```

**修复**:
```python
def _handle_right_click(self, click_pos):
    # 确保 click_pos 是 QPointF 类型
    if not isinstance(click_pos, QPointF):
        click_pos = QPointF(click_pos.x(), click_pos.y())
    # ...
```

## ✅ 修复验证

### 第一步：事件过滤器能捕获事件 ✓
**现象**: 应用启动后，右键点击时控制台输出：
```
[DEBUG] MainWindow.eventFilter 捕获右键点击！
[DEBUG] 点击位置 (相对container): (x, y)
[DEBUG] 左canvas几何: x=..., y=..., w=..., h=...
```

**结论**: 事件过滤器成功安装在 viewport() 上，能够捕获右键点击事件。

### 第二步：修复类型不匹配 ✓
**问题**: `QRectF.contains()` 期望 `QPointF` 类型，但 `event.pos()` 返回 `QPoint` 类型

**修复**: 在 `_handle_right_click` 方法中添加类型检查和转换

**预期结果**: 右键点击脱敏框时，应该能成功删除并输出：
```

[DEBUG] 点击位置: (x, y)
[DEBUG] 手动框数量: N
[DEBUG] OCR框数量: M
[DEBUG] ✓ 删除XXX框: 页面0, 索引X
```
