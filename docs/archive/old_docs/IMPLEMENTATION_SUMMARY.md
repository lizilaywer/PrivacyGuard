# Word 文档脱敏功能实现总结

## 已完成的功能

### 1. 文件类型检测
- 支持 `.pdf` 和 `.docx` 格式的自动检测
- 在文件打开对话框中支持多种格式选择
- 实现位置: `main.py:940-950`

### 2. Word 文档打开
- 使用 `python-docx` 库打开 `.docx` 文件
- 初始化 `word_data` 结构，存储段落和表格信息
- 自动切换到 HTML 预览模式
- 隐藏 PDF 特有控件（双页模式、黑白遮罩等）
- 实现位置: `main.py:1000-1056`

### 3. 智能扫描功能
- 实现了 `WordWorker` 线程类进行后台扫描
- 支持段落和表格单元格的敏感信息扫描
- 复用了现有的规则库（身份证号、手机号、邮箱等）
- 提供进度反馈
- 实现位置: `main.py:439-533`

### 4. HTML 预览和高亮
- 使用 `mammoth` 库将 Word 转换为 HTML
- 在 HTML 中用黄色 `<mark>` 标签高亮敏感信息
- 使用 `QWebEngineView` 显示预览
- 实现位置: `main.py:1222-1290`

### 5. 自定义替换文本
- 添加了替换文本输入框
- 仅在打开 Word 文档时显示
- 支持用户自定义脱敏替换文本
- 实现位置: `main.py:646-664` 和 `_on_replacement_changed` 方法

### 6. 格式保持保存（核心功能）
- 使用副本方式修改文档，保留所有格式
- 实现 run 级别的文本替换，保持原有格式属性：
  - 粗体、斜体、下划线、删除线
  - 字体名称、字号、颜色
  - 高亮、下标/上标
- 支持段落和表格的格式保持
- 实现位置: `main.py:1323-1492`

## 使用流程

1. 打开文档 → 选择 .docx 文件
2. 查看预览 → HTML 预览自动显示
3. 设置规则 → 点击"高级设置"选择脱敏规则
4. 自定义替换 → 在"替换文本"输入框输入自定义内容
5. 智能扫描 → 点击"智能脱敏"开始扫描
6. 查看结果 → 敏感信息黄色高亮显示
7. 保存文档 → 点击"导出"保存脱敏后的文档

## 已知限制

1. 跨 run 敏感信息格式保持可能不完美
2. 暂不支持 .doc 格式
3. 复杂表格可能需要手动检查
