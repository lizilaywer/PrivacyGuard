### 3. 修复 Batch 文件换行符 ✅
问题: Batch 文件使用 LF 换行符导致解析错误

修复:
```bash
python3 -c "
import glob
for filepath in glob.glob('*.bat'):
    with open(filepath, 'rb') as f:
        content = f.read()
    content = content.replace(b'\r\n', b'\n').replace(b'\r', b'\n')
    content = content.replace(b'\n', b'\r\n')
    with open(filepath, 'wb') as f:
        f.write(content)
"
```

**结果**: ✅ 修复了 batch 解析错误，但 DLL 问题仍未解决

## 下次尝试方案

### 方案 1: 分析 DLL 依赖
使用 Visual Studio 的 `dumpbin` 工具:
```cmd
dumpbin /dependents onnxruntime_pybind11_state.pyd
```

### 方案 2: 安装 VC++ Redistributable 到打包环境
在打包用的 Windows 机器上安装:
- https://aka.ms/vs/17/release/vc_redist.x64.exe

### 方案 3: 检查 onnxruntime 完整依赖
```python
import onnxruntime
print(onnxruntime.__file__)
# 检查该目录下的所有 DLL
```

### 方案 4: 使用 Dependency Walker
使用 Dependencies 工具分析:
- https://github.com/lucasg/Dependencies

### 方案 5: 尝试不同 onnxruntime 版本
```bash
pip uninstall onnxruntime onnxruntime-gpu
pip install onnxruntime==1.15.1
```

### 方案 6: 使用 --collect-binaries
在 spec 文件中使用更激进的收集方式:
```python
from PyInstaller.utils.hooks import collect_dynamic_libs
binaries = collect_dynamic_libs('onnxruntime', destdir='onnxruntime/capi')
```
